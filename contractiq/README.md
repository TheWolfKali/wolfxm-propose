# ContractIQ

AI-powered construction proposals in under 3 minutes. Contractors take job site photos, describe the scope, and get a professional proposal with line-item estimates generated by Google Gemini — ready to send to homeowners.

## Features

- **AI Estimates** — Gemini 2.5 Flash analyzes job photos + description to generate detailed line-item estimates
- **Editable Estimates** — Adjust line items, labor hours, markup percentage before sending
- **Shareable Proposals** — Generate a PDF and shareable link for the homeowner
- **Homeowner Approval** — Clients approve proposals via a public link (no login required)
- **Real-time Dashboard** — Live status updates when homeowners approve
- **PWA** — Installable on iOS/Android from Chrome/Safari "Add to Home Screen"
- **Mobile-first** — Built for contractors working in the field on their phones

## Tech Stack

- **Framework**: Next.js 16 (App Router)
- **Database + Auth**: Supabase (Postgres + Row Level Security)
- **AI**: Google Gemini 2.5 Flash
- **PDF**: @react-pdf/renderer
- **Styling**: Tailwind CSS v4

## Setup

### 1. Clone and install

```bash
git clone <repo>
cd contractiq
npm install
```

### 2. Create a Supabase project

1. Go to [supabase.com](https://supabase.com) and create a new project
2. Run `supabase-setup.sql` in the SQL Editor (Database → SQL Editor)
3. Run `supabase-triggers.sql` in the SQL Editor (adds approval trigger + avatars bucket)

### 3. Set up environment variables

Create `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
GEMINI_API_KEY=your-gemini-api-key
NEXT_PUBLIC_SITE_URL=https://your-domain.com
```

Get Supabase keys from: Project Settings → API

Get a Gemini API key from: [Google AI Studio](https://aistudio.google.com/app/apikey)

`NEXT_PUBLIC_SITE_URL` is used for proposal share links. In development, leave it unset and the app uses the request origin automatically.

### 4. Run locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

### 5. PWA icons (already generated)

```bash
node scripts/generate-icons.mjs
```

## Deploy to Vercel

1. Push to GitHub
2. Import the repo at [vercel.com](https://vercel.com)
3. Add all environment variables from step 3
4. Deploy — Vercel handles Next.js builds automatically

## Database Schema

| Table | Purpose |
|-------|---------|
| `contractors` | Contractor profiles (1:1 with auth.users) |
| `jobs` | Job records with type, description, status |
| `job_photos` | Photo paths in Supabase Storage |
| `estimates` | AI-generated line-item estimates |
| `proposals` | Shareable PDFs with approval tracking |

## Storage Buckets

| Bucket | Access | Purpose |
|--------|--------|---------|
| `job-photos` | Private (contractor only) | Job site photos |
| `proposals` | Public (read) | Generated PDF proposals |
| `avatars` | Public (read) | Contractor logos |

## User Flow

1. Contractor signs up → sets company name, labor rate, markup
2. Creates a job → selects type → adds photos → describes scope → adds address
3. AI generates estimate with line items, labor hours, warnings
4. Contractor edits estimate → taps "Generate & Send to Homeowner"
5. Share modal appears with link, copy, Web Share API, SMS options
6. Homeowner opens `/proposal/[token]` → reviews → approves
7. Dashboard updates in real-time → job status changes to "approved"
